System Architecture
How the detector works
Five pipeline stages transform raw text into a semantic threat profile — detecting what words alone cannot reveal.
STAGE 01
Input ingestion
Raw text enters the pipeline. No preprocessing that could destroy semantic signal — punctuation, casing, and syntax preserved.
STAGE 02
Semantic embedding
Text is converted to a dense vector in high-dimensional space. Words with similar intent cluster — even if surface forms differ entirely.
STAGE 03
Threat projection
A learned projector layer rotates the embedding space so threat-relevant dimensions become separable from benign signal.
STAGE 04
Cluster proximity
The projected vector is compared against known threat cluster centroids. Distance in semantic space becomes a probability score.
STAGE 05
Signal extraction
Interpretable signals are extracted — attack type, intent trajectory, manipulation pattern, and confidence with human-readable reasoning.
